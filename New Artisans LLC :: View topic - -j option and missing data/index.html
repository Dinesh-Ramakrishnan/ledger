yes
what i mean, is what i have in my .dat file transaction for, lets say Food and they are not strictly periodical ( i mean they made not each day of the period i have them for 1,3,7,11,12,13-th of Jan for example) than gnuplot draws not what i expect, but the uninterrupted chart from the first date of the period till the last one

i'd like to see bars only for the days where the data are present and no bar for the days when no transaction were made.

my shell script to plot the chart is (the positional parameters depends on what you put as options to the ledger):
Code:
#!/bin/sh
#
# report  -j -n -p "daily from 2007.01.01" reg Expenses:Food
#

# last parameter
what=`echo $@|awk '{print arr[split($0,arr)]}'`

period=$4

(cat <<EOF; ledger "$@") | gnuplot
set terminal x11 persist enhanced font 'arial,15'
set encoding koi8r
set locale 'ru_RU.KOI8-R'
#
set title '$what ( $period )' font 'arial,24'
set xrange [-2:2]
set yrange [-2:2]
#
set xtics rotate by -90 font 'arial,10' tc lt 3
set ytics font 'arial,10' tc lt 3
set grid
set auto
set contour
#
set ylabel 'UAH'
#
set key under
set datafile missing "-"
set xdata time
set timefmt "%Y.%m.%d"
set format x "%Y.%m.%d"
#set timestamp 'drawn: %y.%m.%d %H:%M' bottom
#
set style fill solid 0.3 border
#
plot "-" using 1:2 title '$what' with boxes
EOF
